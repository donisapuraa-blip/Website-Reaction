API Documentation

REST API v2
GET /api/v2/status

Retrieve current system status, uptime metrics, and server health indicators.

Headers

Authorization: Bearer {API_KEY}
Content-Type: application/json
X-Request-ID: {uuid}

Response · Success 200

{
  "status": "operational",
  "uptime": "99.98%",
  "latency_ms": 42,
  "timestamp": "2026-06-08T10:00:00Z"
}

Response · Error 401

{
  "error": "Unauthorized",
  "message": "Invalid or expired API key",
  "code": 401
}
POST /api/v2/request

Submit a new API request with target URL, message payload, and optional metadata.

Request Body

{
  "url": "https://target.example.com",
  "message": "Hello World 🚀",
  "method": "POST",
  "metadata": {
    "priority": "high",
    "tags": ["test", "prod"]
  }
}

Response · Success 200

{
  "success": true,
  "request_id": "req_abc123xyz",
  "delivered": true,
  "latency_ms": 187,
  "remaining_limit": 450
}

Response · Error 429

{
  "error": "RateLimitExceeded",
  "message": "Daily request limit reached",
  "retry_after": 3600,
  "code": 429
}

Quick Integration

// JavaScript — Fetch API
const response = await fetch('https://api.nexus.io/v2/request', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    url: 'https://target.example.com',
    message: 'Hello World 🚀'
  })
});

const data = await response.json();
console.log(data);

Your API key is auto-generated when you first open the dashboard and stored securely in your browser's localStorage. It's displayed in the API Key / User ID card on the Overview section. You can regenerate it anytime by clicking the refresh icon on that card.

The free tier allows 500 requests per day. Pro plans start at 10,000 requests/day. Limits reset at midnight UTC. You can monitor your remaining limit in real-time via the Remaining Limit card.

Your API key is stored locally in localStorage and never transmitted unless you explicitly make an API call. For production use, we strongly recommend storing keys in environment variables or a secrets manager rather than in browser storage.
